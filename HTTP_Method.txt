Метод HTTP указывает какое желаемое действие выполнится для данного ресурса.
Обычно название метода представляет собой короткое название записанное заглавными буквами.
Название метода чувствительно к регистру.
Название метода можкт быть любым, но тогда и клиент и сервер должны их знать.
Если API не распознал указанный  метод клиентом, то он должен вернуть статус-код 504 (Not Implemented).
Если API известен метод, но он не применим к конкретному ресурсу, то он возвращает сообщение с кодом 405 (Method Not Allowed). 
Метод может называться как угодно, но есть общепринятые методы.
*GET - запрашивает представлени о ресурсе. Запросы с использованием этого метода могут только извлекать данные. Основные запросы идет методом GET. Сам запрос не имеет тела, но оно содержится в теле ответа.
*HEAD - запрашивает ресурс так же, как и метод GET, но без тела ответа.
*POST - используется для отправки данных опеределенному ресурсу. Этот метод мы используем, если нам что то нужно передать серверу (например логин и пароль).
*PUT - заменяется текущее представление ресурса данными запроса. Отличие от POSTв том, что PUT дает один и тот же результат каждые раз, а POST может создавать каждый раз новые данные.
*DELETE - удаляет указанный ресурс.
*CONNECT - запускает двусторонюю свзяь с запрошенными ресурсами. Метод обычно используется для открытия прозрачного TCP IP тоннеля.
*OPTIONS - используется для описания параметров соеденения с ресурсами.
*TRACE - возвращает полученный запрос так, что клиент может увидеть какую информацию промежуточные серверы добавляют или изменяют в запросе.
*PATCH - используется для частичного изменения ресурса.
